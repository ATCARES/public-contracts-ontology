Recept pro vyznačení veřejné zakázky v RDFa

Autoři:
  • Martin Nečaský (MFF UK)

Pro popis veřejných zakázek pomocí formátu RDFa používáme slovník, který jsme pro tyto účely vytvořili. Slovník kombinuje některé všeobecně uznávané slovníky, jako např. GoodRelations nebo Dublin Core.

HTML vs. XHTML

Prozatím je standardizováno pouze rozšíření jazyka XHTML o prvky jazyka RDFa. XHTML dokument potom vypadá takto:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML+RDFa 1.1//EN" "http://www.w3.org/MarkUp/DTD/xhtml-rdfa-2.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="cs" lang="cs" version="XHTML+RDFa 1.1">
   …
<html>

Pokud potřebujeme použít HTML, je to možné. RDFa rozšíření jazyka HTML ještě sice není standardizováno, ale fungovat bude. HTML dokument vypadá takto:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01+RDFa 1.1//EN"
   "http://www.w3.org/MarkUp/DTD/html401-rdfa11-1.dtd">
<html>
   …
</html>

Jmenné prostory

Nejprve je vždy nutné zavést potřebné jmenné prostory. To můžeme udělat v jakémkoliv elementu HTML stránky, který obsahuje RDFa kód. Jmenné prostory jsou pak platné v celém obsahu tohoto elementu.

Pro kompatibilitu s novější verzí RDFa 1.1 a XHTML+RDFa 1.1 je potřeba zavádět jmenné prostory pomocí atributu prefix

<html xmlns="http://www.w3.org/1999/xhtml"
      prefix="rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns#
              rdfs: http://www.w3.org/2000/01/rdf-schema#
              gr: http://purl.org/goodrelations/v1#
              dc: http://purl.org/dc/terms/
              pc: http://purl.org/opendata-cz/public-contracts#
              payment: http://reference.data.gov.uk/def/payment#
              xsd: http://www.w3.org/2001/XMLSchema#">
   …
</html>

Pro kompatibilitu se starší verzí RDFa 1.0 a XHTML+RDFa 1.0 je potřeba zavádět jmenné prostory pomocí XML jmenných prostorů

<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
      xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"
      xmlns:gr="http://purl.org/goodrelations/v1#"
      xmlns:dc="http://purl.org/dc/terms/"
      xmlns:pc="http://purl.org/opendata-cz/public-contracts#"
      xmlns:payment="http://reference.data.gov.uk/def/payment#"
      xmlns:xsd="http://www.w3.org/2001/XMLSchema#">
   …
</html>

Pro kompatibilitu s oběma verzemi je možné oba způsoby zavedení jmenných prostorů použít najednou.

Určení místa na webové stránce, kde je veřejná zakázka popsána

Informace o veřejné zakázce uzavřete do jednoho elementu, např. div. Potom určete, že obsah tohoto elementu se týká veřejné zakázky pomocí RDFa atributu typeof s hodnotou pc:Contract. Jinými slovy tím říkáte, že obsah reprezentuje instanci třídy pc:Contract.

<div typeof="pc:Contract">
  …
</div>

Výše uvedeným dále říkáte, že samotná veřejná zakázka je anonymní, tj. nemá žádný identifikátor. To je v pořádku do té doby, kdy se chcete na zakázku odkazovat. Pokud chcete zakázce identifikátor přiřadit (např. proto, abyste se na ní mohli odkazovat), je potřeba jí přiřadit jednoznačné URI (v RDF se jako identifikátory používají právě URI). Toto URI může být např. fyzickým URL, na kterém je umístěna webová stránka s veřejnou zakázkou. Je důležité ale zajistit, že zvolené identifikační URI se pro danou veřejnou zakázku již nikdy nezmění. Pokud toto nemůžete u fyzického URL zajistit, je vhodnější zvolit nějaké virtuální URI (např. URL vašeho webu doplněné o /VZ/<cislo-jednaci>), na kterém fyzicky žádný dokument existovat nemusí.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
</div>

Údaje při zveřejnění veřejné zakázky

Pro každou veřejnou zakázku uvádějte její název. Pro názvy je v našem slovníku určena RDF vlastnost dc:title.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   Název: <span property="dc:title">Jezdecké potřeby městské policie</span>
</div>

Dále uvádějte datum zveřejnění a datum pro posílání nabídek. Pro ně jsou v našem slovníku určeny RDF vlastnosti dc:issued a pc:bidDeadline. Je zde ale jeden podstatný rozdíl oproti názvu. Na webové stránce chcete návštěvníkům zobrazit datum ve formátu běžném v ČR, např. 1. 2. 2010 nebo 1. února 2010. To ale není strojově čitelné datum. Pro strojově čitelnou podobu dat používáme datový typ xsd:date jazyka XML Schema, např. 2010-02-01 (RRRR-MM-DD). Strojově čitelnou hodnotu ale pravděpodobně nechcete ukazovat svým návštěvníkům. Proto ji dávejte jako hodnotu atributu content. Také přidejte informaci o datovém typu pomocí atributu datatype.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Datum zveřejnění: <span property="dc:issued" content="2010-01-01" datatype="xsd:date">1. 1. 2010</span>
   Datum pro podávání nabídek: <span property="pc:bidDeadline" content="2010-02-01" datatype="xsd:date">1. 2. 2010</span>
</div>

Dalším údajem je předmět veřejné zakázky ve formě CPV kódu. Seznam CPV kódů platných od roku 2008 je na adrese http://opendata.cz/cpv/2008/. Pro vyhledávání CPV kódů v rámci svého softwarového řešení můžete použít jednoduchý nástroj http://opendata.cz/cpv/. Předmětů může mít veřejná zakázka více. Zadávejte je pomocí RDF vlastnosti pc:subject.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Předmět: <span rel="pc:subject" resource="http://opendata.cz/cpv/2008/#cpv-18911000-8" datatype="xsd:string">sedla</span>
   <span rel="pc:subject" resource="http://opendata.cz/cpv/2008/#cpv-18913000-2" datatype="xsd:string">biče</span>
</div>

Údaje při zadání veřejné zakázky

Po zadání veřejné zakázky můžete zveřejnit další údaje. První informací je počet obdržených nabídek, k čemuž použijte RDF vlastnost pc:numberOfBids.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   Počet obdržených nabídek: <span property="pc:numberOfBids" datatype="xsd:int">2</span>
</div>

Slovník umožňuje také zveřejnit jednotlivé nabídky jako instance třídy pc:Bid. Nabídku přiřadíte k veřejné zakázce pomocí vlastnosti pc:bid. Nabídek můžete přiřadit několik.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
      …
  </div>
  <div rel="pc:bid">
      …
  </div>
</div>

Samotnou nabídku umístěte do samostatného elementu (např. div) s RDFa atributem typeof s hodnotou pc:Bid.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
</div>

Ke každé nabídce přidejte společnost, která zaslala nabídku, pomocí vlastnosti pc:bidder. Společnost uvádějte jako anonymní instanci třídy gr:BusinessEntity, tj. v samostatném elementu s RDFa atributem typeof s hodnotou gr:BusinessEntity.

<div about="http://www.mesto.cz/VZ/12345678" typeof="c:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          …                 	
        </div>
      </div>
    </div>
  </div>
  <div rel="pc:bid">
    …
  </div>
</div>

Doplňte data o nabízejícím - jeho název (RDF vlastnost gr:legalName) a IČO (RDF vlastnost pc:orgNumber).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
    </div>
  </div>
</div>

Kromě nabízejícího uvádějte k nabídce také nabídnutou cenu pomocí vlastnosti pc:bidPrice. Cenu uvádějte jako anonymní instanci třídy gr:PriceSpecification, tj. v samostatném elementu s RDFa atributem typeof s hodnotou gr:PriceSpecification.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
      <div rel="pc:bidPrice">
        <div typeof="gr:PriceSpecification">Nabídnutá cena:
          …
        </div>
      </div>
    </div>
  </div>
</div>

K ceně doplňte hodnotu (RDF vlastnost gr:hasCurrencyValue), měnu (RDF vlastnost gr:hasCurrency) a zda je cena s DPH (RDF vlastnost gr:valueAddedTaxIncluded).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Nabídky:
  <div rel="pc:bid">
    <div typeof="pc:Bid">
      <div rel="pc:bidder">
        <div typeof="gr:BusinessEntity">Nabízející:
          <span property="gr:legalName">První jezdecká a.s.</span>
          , IČ
          <span property="pc:orgNumber">789654123</span>                 	
        </div>
      </div>
      <div rel="pc:bidPrice">
        <div typeof="gr:PriceSpecification">Nabídnutá cena:
          <span property="gr:hasCurrencyValue" datatype="xsd:int">1.500.000</span>
          <span property="gr:hasCurrency" content="CZK">Kč</span>
          <span property="gr:valueAddedTaxIncluded" content="false" datatype="xsd:boolean">bez daně</span>
        </div>
      </div>
    </div>
  </div>
</div>

Vítěznou nabídku nepřiřazujte k veřejné zakázce pomocí RDF vlastnosti pc:bid ale pomocí vlastnosti pc:acceptedBid. Samotnou vítěznou nabídku popište stejně jako ostatní nabídky, tj. jako instance třídy pc:Bid.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Vítězná nabídka:
  <div rel="pc:acceptedBid">
    <div typeof="pc:Bid">
      …
    </div>
  </div>
</div>

K veřejné zakázce můžete uvést dokumenty, které se ji týkají (zadávací dokumentace, protokol z rozbalování obálek s nabídkami, smlouva).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
   …
   <div><a rel="pc:document" href="http://www.mesto.cz/docs/12345678_popis.doc">Popis zakázky (DOC)</a></div>
   <div><a rel="pc:document" href="http://www.mesto.cz/docs/12345678_smlouva.pdf">Smlouva (PDF)</a></div>
</div>

Cenu za zakázku dohodnutou s vítězným dodavatelem na smlouvě můžete uvést pomocí vlastnosti pc:contractPrice (opět jako instanci třídy gr:PriceSpecification).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  <div rel="pc:contractPrice">
    <div typeof="gr:PriceSpecification">Smluvní cena:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">1.000.000</span>
      <span property="gr:hasCurrency" content="CZK">Kč</span>
      (
      <span property="gr:valueAddedTaxIncluded" content="false" datatype="xsd:boolean">bez daně</span>
      )
    </div>
  </div>
</div>

Údaje při dokončení realizace veřejné zakázky

Dodatkovou cenu (cena navíc vzniklá v průběhu realizace zakázky) a dotaci (částka z celkové konečné ceny zakázky, která byla pokryta nějakou dotací) můžete uvést pomocí vlastnosti c:supplementPrice, resp. pc:subsidy (opět jako instance třídy gr:PriceSpecification).

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  <div rel="pc:supplementPrice">
    <div typeof="gr:PriceSpecification">Dodatková cena:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">150.000</span>
      <span property="gr:hasCurrency" content="CZK">Kč</span>
    </div>
  </div>
  <div rel="pc:subsidy">
    <div typeof="gr:PriceSpecification">Dotace:
      <span property="gr:hasCurrencyValue" datatype="xsd:int">20.000</span>
      <span property="gr:hasCurrency" content="EUR">EUR</span>
    </div>
  </div>
</div>

Také můžete pomocí vlastností pc:quality a pc:onSchedule uvést, zda byla splněna očekávaná kvalita při realizaci veřejné zakázky a zda byly dodrženy termíny.

<div about="http://www.mesto.cz/VZ/12345678" typeof="pc:Contract">
  …
  Splněna v termínu: <span property="pc:onSchedule" content="false" datatype="xsd:boolean">ano</span>
  Splněna požadovaná kvalita: <span property="pc:quality" content="true" datatype="xsd:boolean">ne</span>
</div>
